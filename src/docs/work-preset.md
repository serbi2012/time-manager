# 작업 프리셋

**작업 프리셋**은 자주 수행하는 작업을 미리 저장해두고 빠르게 추가할 수 있는 기능이에요. 매번 작업 정보를 입력하는 번거로움 없이 **+** 버튼 한 번으로 작업을 추가할 수 있어요.

---

## 프리셋의 이점

| 이점               | 설명                                   |
| ------------------ | -------------------------------------- |
| **빠른 작업 추가** | + 버튼 한 번으로 작업 추가             |
| **일관된 데이터**  | 항상 동일한 프로젝트 코드, 작업명 사용 |
| **시간 절약**      | 반복 입력 불필요                       |
| **정렬 관리**      | 자주 쓰는 프리셋을 상단에 배치         |

### 위치

-   **데스크톱**: [일간 기록](wiki:daily-record) 페이지의 좌측 사이드바 (300px)
-   **모바일**: 우측 하단 FAB 버튼 → 하단 프리셋 드로어

### 프리셋 카드 구성

| 요소                     | 설명                                             |
| ------------------------ | ------------------------------------------------ |
| **왼쪽 색상 스트라이프** | 프리셋 구분용 세로 색상 바 (3px)                 |
| **드래그 핸들**          | 호버 시 나타나는 아이콘, 드래그하여 순서 변경    |
| **제목**                 | 거래명(있으면) 또는 작업명                       |
| **메타 정보**            | 작업명 뱃지(거래명이 있을 때), 업무명 + 카테고리 |
| **더보기 메뉴**          | 수정/삭제 드롭다운                               |
| **+ 버튼**               | 작업 추가 (툴팁: "작업 추가")                    |

### 빈 상태

프리셋이 하나도 없으면 다음 메시지가 표시돼요:

-   **"아직 프리셋이 없어요"**
-   "자주 쓰는 작업을 저장해 보세요"

## 프리셋 추가

### 추가 방법

1. 사이드바 상단의 **"새 프리셋 추가"** 버튼 클릭 (단축키: `Alt + P`)
2. 프리셋 추가 모달에서 정보 입력
3. **"추가 (F8)"** 버튼 클릭

### 입력 필드

| 필드              | 설명               | 필수 | 예시            |
| ----------------- | ------------------ | :--: | --------------- |
| **프로젝트 코드** | 프로젝트 식별 코드 |  -   | `A25_01846`     |
| **작업명**        | 프로젝트/업무명    |  O   | `프레임워크 FE` |
| **업무명**        | 업무 유형          |  -   | `개발`          |
| **거래명**        | 작업 상세 제목     |  -   | `컴포넌트 개발` |
| **카테고리**      | 작업 분류          |  -   | `개발`          |
| **비고**          | 추가 메모          |  -   | -               |
| **색상**          | 구분용 색상 선택   |  O   | -               |

## 프리셋 수정 및 삭제

### 수정하기

1. 프리셋 카드의 **더보기 메뉴 (...)** 클릭
2. **"수정"** 선택
3. 수정 모달에서 내용 변경
4. **"저장 (F8)"** 버튼 클릭

> 프리셋을 수정해도 **이미 추가된 작업 기록은 변경되지 않아요**.

### 삭제하기

1. 프리셋 카드의 **더보기 메뉴 (...)** 클릭
2. **"삭제"** 선택
3. 확인 팝업에서 **"삭제"** 클릭

## 드래그앤드롭 순서 변경

자주 사용하는 프리셋을 상단에 배치하면 접근성이 좋아져요.

### 순서 변경 방법

```mermaid
flowchart LR
    A[드래그 핸들 클릭] --> B[원하는 위치로 드래그]
    B --> C[마우스 버튼 놓기]
    C --> D[자동 저장]
```

변경된 순서는 **자동으로 저장**되며, 로그인 상태에서는 클라우드에도 동기화돼요.

## 프리셋으로 작업 추가

### + 버튼 사용

프리셋 카드의 **+ 버튼**을 클릭하면 해당 프리셋의 정보로 새 작업이 생성돼요. 추가 완료 시 체크 아이콘이 잠시 표시된 후 원래 + 아이콘으로 돌아와요.

### 거래명 처리

[설정](wiki:settings)의 데이터 탭에서 "고유 식별자 자동 추가" 옵션에 따라 동작이 달라져요:

| 설정           | 동작                                                | 예시                             |
| -------------- | --------------------------------------------------- | -------------------------------- |
| **OFF** (기본) | 이름 그대로 추가, 동명 미완료 작업 있으면 번호 부여 | `회의` → `회의 (2)` → `회의 (3)` |
| **ON**         | 거래명에 타임스탬프 구분자 추가                     | `회의_0122_093045_123`           |

### 타이머 자동 시작 여부

프리셋의 + 버튼은 **작업만 추가**하고 타이머는 시작하지 않아요. 타이머까지 시작하려면 [일간 기록](wiki:daily-record) 테이블에서 해당 작업의 타이머 아이콘을 클릭하세요.

## 활용 팁

### 프로젝트별 프리셋 구성

| 프리셋             | 색상   | 용도             |
| ------------------ | ------ | ---------------- |
| [프로젝트A] FE개발 | 파란색 | 프론트엔드 작업  |
| [프로젝트A] BE개발 | 파란색 | 백엔드 작업      |
| [프로젝트B] 설계   | 녹색   | 설계 문서 작성   |
| [공통] 문서작업    | 노란색 | 일반 문서 작업   |
| [공통] 회의        | 보라색 | 정기/비정기 회의 |

### 순서 정리 팁

-   **자주 사용하는 프리셋**을 상단에 배치
-   **프로젝트 단위**로 그룹화하여 배치
-   완료된 프로젝트의 프리셋은 삭제하거나 하단으로 이동
