# 작업 프리셋

**작업 프리셋**은 자주 수행하는 작업을 미리 저장해두고 빠르게 시작할 수 있는 기능입니다. 매번 작업 정보를 입력하는 번거로움 없이 클릭 한 번으로 작업을 추가할 수 있습니다.

---

## 프리셋의 이점

| 이점               | 설명                                   |
| ------------------ | -------------------------------------- |
| **빠른 작업 시작** | 클릭 한 번으로 작업 추가               |
| **일관된 데이터**  | 항상 동일한 프로젝트 코드, 작업명 사용 |
| **시간 절약**      | 반복 입력 불필요                       |
| **정렬 관리**      | 자주 쓰는 프리셋을 상단에 배치         |

### 프리셋 카드 구성

| 요소                  | 설명                                     |
| --------------------- | ---------------------------------------- |
| **왼쪽 색상 바**      | 프리셋 색상으로 구분되는 세로 스트라이프 |
| **≡ 드래그 핸들**     | 호버 시 표시, 드래그하여 순서 변경       |
| **제목**              | 거래명(있으면) 또는 작업명이 표시됨      |
| **업무명 · 카테고리** | 부가 정보                                |
| **⋯ 더보기**          | 호버 시 표시, 수정/삭제 메뉴             |
| **+ 버튼**            | 원형 아이콘 버튼, 작업 추가              |

## 프리셋 추가

### 추가 방법

1. 사이드바 상단의 **"+ 추가"** 버튼 클릭 (단축키: `Alt + P`)
2. 프리셋 추가 모달에서 정보 입력
3. **"추가"** 버튼 클릭

### 입력 필드

| 필드              | 설명               | 필수 | 예시            |
| ----------------- | ------------------ | :--: | --------------- |
| **프로젝트 코드** | 프로젝트 식별 코드 |  -   | `A25_01846`     |
| **작업명**        | 프로젝트/업무명    |  ✅  | `프레임워크 FE` |
| **업무명**        | 업무 유형          |  -   | `개발`          |
| **거래명**        | 작업 상세 제목     |  -   | `컴포넌트 개발` |
| **카테고리**      | 작업 분류          |  -   | `개발`          |
| **비고**          | 추가 메모          |  -   | -               |
| **색상**          | 구분용 색상 선택   |  ✅  | 🔵              |

### 색상 선택

10가지 색상 중에서 프리셋을 구분할 색상을 선택합니다:

| 색상      | 코드    | 권장 용도          |
| --------- | ------- | ------------------ |
| 🔵 파란색 | #1890ff | 개발 작업          |
| 🟢 녹색   | #52c41a | 테스트, 완료 관련  |
| 🟡 노란색 | #faad14 | 문서, 기획 작업    |
| 🔴 빨간색 | #f5222d | 긴급, 중요 작업    |
| 🟣 보라색 | #722ed1 | 회의, 커뮤니케이션 |

## 프리셋 수정 및 삭제

### 수정하기

1. 프리셋 카드에 마우스를 올림 (호버)
2. 나타나는 **✏️ 수정** 버튼 클릭
3. 수정 모달에서 내용 변경
4. **"저장"** 버튼 클릭

> 프리셋을 수정해도 **이미 추가된 작업 기록은 변경되지 않습니다**.

### 삭제하기

1. 프리셋 카드에 마우스를 올림
2. **🗑️ 삭제** 버튼 클릭
3. 확인 팝업에서 **"삭제"** 클릭

## 드래그앤드롭 순서 변경

자주 사용하는 프리셋을 상단에 배치하면 접근성이 좋아집니다.

### 순서 변경 방법

```mermaid
flowchart LR
    A[≡ 드래그 핸들 클릭] --> B[원하는 위치로 드래그]
    B --> C[마우스 버튼 놓기]
    C --> D[자동 저장]
```

변경된 순서는 **자동으로 저장**되며, 로그인 상태에서는 클라우드에도 동기화됩니다.

## 프리셋으로 작업 추가

### + 버튼 사용

프리셋 카드의 **+ 버튼**을 클릭하면 해당 프리셋의 정보로 새 작업이 생성됩니다.

### 고유 ID 생성

같은 프리셋으로 여러 작업을 생성할 수 있도록, 거래명에 고유 식별자가 자동 추가됩니다:

| 형식                              | 예시                            |
| --------------------------------- | ------------------------------- |
| `{거래명}_{월일}_{시분초}_{랜덤}` | `컴포넌트 개발_0121_143025_847` |

### 타이머 자동 시작 여부

프리셋의 [+] 버튼은 **작업만 추가**하고 타이머는 시작하지 않습니다. 타이머까지 시작하려면 [일간 기록](wiki:daily-record) 테이블의 **"추가 후 시작"** 버튼을 사용하세요.

## 활용 팁

### 프로젝트별 프리셋 구성

| 프리셋             | 색상 | 용도             |
| ------------------ | ---- | ---------------- |
| [프로젝트A] FE개발 | 🔵   | 프론트엔드 작업  |
| [프로젝트A] BE개발 | 🔵   | 백엔드 작업      |
| [프로젝트B] 설계   | 🟢   | 설계 문서 작성   |
| [공통] 문서작업    | 🟡   | 일반 문서 작업   |
| [공통] 회의        | 🟣   | 정기/비정기 회의 |

### 순서 정리 팁

-   **자주 사용하는 프리셋**을 상단에 배치
-   **프로젝트 단위**로 그룹화하여 배치
-   완료된 프로젝트의 프리셋은 삭제하거나 하단으로 이동
